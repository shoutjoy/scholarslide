# ScholarSlide AI — 코드 점검 및 기능 요약

## 코드 점검 결과 및 수정 사항

### 1. 수정한 버그 (index.js)

1. **ReferenceStore 미구현**  
   - `ReferenceStore` IIFE 안에 메서드(`add`, `remove`, `getAll`, `clear`)와 `return`이 없어 참고문헌 추가/삭제가 동작하지 않을 수 있었습니다.  
   - **조치:** `js/store/references.js`와 동일한 구현을 `index.js`에 넣었습니다.

2. **인용 포맷터·저장 목록 미정의**  
   - `formatCitation`, `formatInText`, `getSavedRefList`, `saveRefList`, `addToSavedList`가 `index.js`에 없고, HTML은 `index.js`만 불러옵니다.  
   - **조치:** APA/Chicago/MLA 포맷과 저장 목록 함수를 `index.js`에 추가했습니다.

3. **parseAPA / addParsedRef 누락**  
   - 참고문헌 모달의 "APA 파싱" 탭에서 `parseAPA()`와 `addParsedRef()`를 사용하는데, `index.js`에 정의가 없었습니다.  
   - **조치:** `parseAPA()`와 `addParsedRef()`를 `index.js`에 구현해 두었습니다.

4. **전역 상수·상태 변수 누락**  
   - `LS_ACTIVE_KEY`, `LS_KEYS_LIST`, `LS_SESSIONS`, `LS_SAVED_REF_LIST`와 `rawText`, `slides`, `slideStyle` 등이 `index.js`에 선언되어 있지 않았습니다. (다른 js 파일에만 존재)  
   - **조치:** `index.js` 상단 STATE 블록에 위 상수와 전역 상태 변수 선언을 추가했습니다.

이제 **index.html에서 index.js만 로드해도** 참고문헌·세션·자동저장·API 키 저장이 정상 동작하도록 맞춰 두었습니다.

## 좌측 사이드바: 요약 탭 vs 원고 탭

좌측 패널에는 **원문**, **요약**, **원고** 세 탭이 있습니다. 요약과 원고는 역할이 다르며, **요약 히스토리**와 **원고(생성) 히스토리**는 서로 공유하지 않습니다.

### 요약 탭

- **전체 요약**  
  「전체 요약」을 누르면 **요약 옵션 모달**이 열립니다. 요약 대상(원문/번역), 세밀도(세밀한/기본/핵심), 요약 방식(세밀한/핵심/슬라이드 요약/구조/개념/비교/연구)을 선택한 뒤 「요약 실행」을 누르면 **요약만** 수행되고, 결과는 **현재 요약** 영역과 **요약 히스토리**에 저장됩니다.
- **슬라이드 요약**  
  「슬라이드 요약」을 누르면 같은 **요약 옵션 모달**이 열립니다. 여기서도 요약 방식(예: 슬라이드 요약) 등을 선택하고 실행하면 **요약만** 수행되며, **요약 히스토리**에만 저장됩니다. **실제 슬라이드(JSON)는 생성하지 않습니다.**  
  → 실제 슬라이드를 만들려면 **원고 탭**의 「슬라이드 생성」을 사용합니다.
- **요약 히스토리**  
  「요약 히스토리」 탭에서 과거 요약 목록을 보고, 항목을 선택하면 해당 요약이 「현재 요약」으로 불러와집니다. 이 히스토리는 **원고 탭의 생성 히스토리와 별도**입니다.

### 원고 탭

- **발표원고생성**  
  이미 만든 슬라이드가 있을 때, 각 슬라이드에 맞는 **발표 원고(스크립트)**를 AI로 생성합니다. 완료 후 **원고(생성) 히스토리**에 기록됩니다.
- **슬라이드 생성**  
  슬라이드 수·표지 포함·문체·슬라이드 생성 유형 등을 반영해 **실제 슬라이드 JSON**을 생성합니다. 완료 후 **원고(생성) 히스토리**에 기록됩니다.
- **커스텀 프롬프트**, **슬라이드 원고로 생성**, **파일선택**, **텍스트입력창**  
  원고/슬라이드 생성 시 참고할 지시사항 입력, 다른 파일 열기, 메모용 텍스트 입력이 가능합니다.
- **발표원고 / 슬라이드생성**  
  생성된 발표 원고 목록 보기와 슬라이드 생성 안내 뷰를 전환합니다.
- **생성내용 / 생성히스토리 / 새창보기**  
  생성 결과 표시, **원고(생성) 히스토리** 목록(발표 원고·슬라이드 생성 완료 시점), 발표 원고 새 창 보기를 사용할 수 있습니다.  
  → 이 **생성 히스토리**는 요약 탭의 **요약 히스토리**와 **공유하지 않습니다.**

### 정리

| 구분 | 요약 탭 | 원고 탭 |
|------|--------|--------|
| 전체 요약 / 슬라이드 요약 | 요약만 실행, **요약 히스토리**에 저장 | — |
| 슬라이드 생성 | — | **실제 슬라이드 JSON** 생성, **원고 히스토리**에 저장 |
| 발표 원고 생성 | — | 기존 슬라이드 기준 발표 원고 생성, **원고 히스토리**에 저장 |
| 히스토리 | 요약 히스토리만 표시·저장 | 원고(생성) 히스토리만 표시·저장 |

| **번역** | 요약/원문을 한국어로 번역(캐시 유지), 번역 결과 팝업 보기·복사 |

## 프로젝트 구조 참고

- **index.html** + **index.js** 한 세트로 동작하도록 수정했으며, `index.html`은 `index.js`만 로드합니다.
- `js/` 폴더의 `state/state.js`, `store/references.js`, `utils/citations.js` 등은 **현재 HTML에서 로드되지 않습니다.**  
  빌드/번들로 합치는 구조가 아니라면, 지금은 **index.js에 필요한 내용을 반영한 상태**가 기준입니다.
